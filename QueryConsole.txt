The architecture below is optimized for:

* local LLMs via Ollama
* graph traversal
* hybrid retrieval
* low-latency streaming
* explainable responses
* scalable enterprise workflows

# Intelligent Query Planning

## Example

### User asks:

> "Show customers connected to delayed shipments from suppliers in Asia"

The planner should:

### Step 1 — Entity Extraction

```text id="8a6g3e"
Entities:
- customers
- delayed shipments
- suppliers
- Asia
```

### Step 2 — Graph Expansion

```text id="ukf7l8"
Customer
  → Order
      → Shipment
          → Supplier
              → Region
```

### Step 3 — Cypher Generation

```cypher
MATCH (c:Customer)-[:PLACED]->(o:Order)
      -[:HAS_SHIPMENT]->(s:Shipment)
      -[:SUPPLIED_BY]->(sup:Supplier)
WHERE s.status = 'Delayed'
AND sup.region = 'Asia'
RETURN c, s, sup
```

### Step 4 — Context Expansion

Retrieve:

* shipment metadata
* supplier risk scores
* historical delays
* related entities

### Step 5 — LLM Synthesis

Generate:

* explanation
* trends
* recommendations
* risk summary
